# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A full-stack NFT marketplace on **BNB Smart Chain** used for 60-minute technical hiring assessments. Four independent sub-projects exist, one per role. Each has pre-built infrastructure and stubbed code that candidates implement.

## Common Commands

```bash
# Install all dependencies
make install

# Start services individually (each in its own terminal)
cd contracts && npx hardhat node                                    # Local blockchain (31337)
cd contracts && npx hardhat run scripts/deploy.js --network localhost # Deploy contracts
cd backend  && npm run dev                                          # Backend API on :8080
cd payments && npm start                                            # Payments service on :3003
cd frontend && npm run dev                                          # Frontend on :3000

# Or start all via Docker
make dev  # docker-compose up --build

# Run all tests
make test

# Run tests per project
cd contracts && npx hardhat test
cd backend   && npm test          # node --test test/**/*.test.js
cd payments  && npm test          # node --test test/**/*.test.js
cd frontend  && npm run lint      # ESLint (no unit test suite)

# Run a single backend/payments test file
cd backend  && node --test test/marketplace.test.js
cd payments && node --test test/settlement.test.js

# Contracts
cd contracts && npx hardhat compile
cd contracts && npx hardhat test test/Staking.test.js  # Single test file
```

## Architecture

### Monorepo with 4 independent sub-projects

Each folder is a self-contained npm project with its own `package.json`. There is no workspace or shared dependency management.

| Folder | Stack | Port | Module System |
|--------|-------|------|---------------|
| `contracts/` | Solidity 0.8.20, Hardhat, OpenZeppelin v5 | 8545 (hardhat node) | CommonJS |
| `backend/` | Node.js, Express | 8080 | CommonJS |
| `frontend/` | React 18, TypeScript, Vite, Wagmi, RainbowKit, Tailwind CSS v3 | 3000 | ESM |
| `payments/` | Node.js, Express | 3003 | CommonJS |

### What's implemented vs stubbed

**Contracts** — `Staking.sol` is stubbed (empty `stake()`, `unstake()`, `pendingRewards()`, `getStakedTokens()`). The other three contracts (`NFTToken.sol`, `RewardToken.sol`, `NFTMarketplace.sol`) are fully implemented. Tests in `test/Staking.test.js` are all skipped, waiting for candidate implementation.

**Backend** — Three routes return 501 in `server.js` (~lines 92-114): `GET /api/v1/nfts/:id/history`, `POST /api/v1/nfts/:id/bid`, `GET /api/v1/transactions/user/:address`. All other routes work. Data is in-memory via `data/seedData.js` (10 NFTs, 3 transactions, generated history). Tests in `test/marketplace.test.js` have 4 stubbed test cases.

**Payments** — `src/settlement.js` has two stubbed functions: `validatePayment(event)` (2% slippage check) and `triggerFulfillment(orderId)` (idempotent fulfillment with retry safety). `lib/fulfillmentApi.js` is the mock API they call into (fully implemented; order IDs ending in `_FAIL` throw). Tests in `test/settlement.test.js` have 7 stubbed cases.

**Frontend** — Fully built. Candidates add wallet connect, buy/stake interactions, and UI features. Key files: `src/pages/NFTDetail.tsx` (buy/stake/bid actions), `src/pages/VerifyDashboard.tsx` (assessment verification UI with 4 role tabs).

### Verification dashboard

`http://localhost:3000/verify` — A single page with 4 tabs (one per role) that live-checks whether each assessment feature is working. Green checkmarks = passing, red X = failing. This is the primary way candidates confirm their work.

### Key data flow

- Frontend calls backend API (`VITE_API_URL`, default `http://localhost:8080/api/v1`) via axios (`src/utils/api.ts`)
- Frontend calls payments service (`VITE_PAYMENTS_URL`, default `http://localhost:3003`) for webhook simulation
- Frontend interacts with smart contracts via wagmi/viem using ABIs in `src/config/contracts.ts`
- Contract addresses come from env vars or `contracts/deployed-addresses.json` (generated by deploy script)

### Networks

- **BSC Testnet**: chainId 97, RPC `https://data-seed-prebsc-1-s1.bnb.org:8545/`
- **Hardhat Localhost**: chainId 31337, RPC `http://127.0.0.1:8545`

## Code Style

- 2-space indentation for all file types (`.editorconfig`)
- LF line endings
- Backend/payments use CommonJS (`require`/`module.exports`)
- Frontend uses ESM with TypeScript
- Backend/payments tests use Node.js built-in test runner (`node --test`)
- Contracts tests use Hardhat's Mocha-based runner with ethers.js
